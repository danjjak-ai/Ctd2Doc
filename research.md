# Research: Multilingual Medical Document Fine-Tuning with Gemma 2

## 1. 도메인 및 데이터 소스 분석
### 1.1 CTD (Common Technical Document)
- **특징:** 의약품 허가 신청을 위한 국제 공통 서식. 대량의 임상 통계 수치, 부작용 데이터, 화학적 조성 포함.
- **포맷 및 언어:** 주로 PDF 포맷. 글로벌 제약사는 영어(FDA/EMA 제출용), 일본 허가용은 일본어(PMDA 제출용)가 주류.
- **추출 챌린지:** 수많은 복잡한 표(Table) 구조와 계층적 헤더 구조 파싱 필요.

### 1.2 일본 의약품 전문 출력 문서 (정답 데이터)
- **의약품 인터뷰 폼 (医薬品インタビューフォーム, IF):** 일본병원약제사회에서 규정한 양식으로, 의사/약사 등 전문가용 상세 설명서. 구조화가 매우 엄격함.
- **쿠스리노 시오리 (くすりのしおり):** 일반인/환자용 복약 안내서. 어려운 의학 용어를 극도로 쉬운 일상어(일본어 가타카나/히라가나 혼용)로 변환한 것이 특징. 포맷이 정형화되어 있음.
- **수집 출처:** 일본 PMDA(의약품의료기기総合機構) 및 RAD-AR(くすりの適正使用協議会) 오픈 데이터 허브.

## 2. 핵심 기술 챌린지 및 솔루션
### 2.1 PDF 표(Table) 구조 유실 문제
- **현상:** 일반 PDF-to-Text 라이브러리는 표 내부의 행/열 관계를 무시하고 텍스트를 파괴함. 임상 수치 매칭률 저하의 주원인.
- **솔루션:** `Marker` 또는 `Nougat`를 활용하여 모든 표 구조를 Markdown Table format(`|---|---|`)으로 변환 보존.

### 2.2 고유 명사 및 수치 왜곡 (Hallucination)
- **현상:** LLM 가중치 학습(LoRA)만으로는 미묘한 용량 수치(e.g., 0.5mg vs 5.0mg)나 부작용 발생률(e.g., 0.1%)의 환각을 완벽히 통제할 수 없음.
- **솔루션:** Retrieval-Augmented Generation(RAG)을 통한 '컨텍스트 강제 주입형 SFT' 데이터셋 구축.

### 2.3 자율 피드백 루프 (Autonomous Feedback Loop)
- **현상:** 정답 문서와 생성 문서 간의 단순 BLEU/ROUGE 스코어는 의료 문서의 '정확성'을 대변하지 못함. (용어 변경이나 의미 왜곡을 잡아내지 못함)
- **솔루션:** **LLM-as-a-Judge (Gemma 2 27B-IT 또는 GPT-4o-mini API)** 시스템을 도입하여 3대 지표(수치 정확성, 포맷 준수율, 가독성/언어 뉘앙스)를 정량 점수화(0~100)하여 보상 스코어로 환산.

## 3. GPU 자원 최적화 (Google Colab 환경)
- **모델:** `google/gemma-2-9b-it` (영어/일본어 도메인 성능 최상위)
- **구동 환경:** Colab Pro (A100 40GB VRAM 또는 L4 24GB VRAM)
- **메모리 절약:** `Unsloth` 프레임워크를 이용한 4-bit QLoRA 적용. 맥락 수용 길이(Context Window)를 4,096까지 확보하여 CTD 청크 다량 주입 가능.
